 DATABASE STRUCTURE (placement_tracker_db)


1  users (Main user collection)
     id, email (unique), password (BCrypt hashed)
     fullName, role (STUDENT/MENTOR)
     departmentId, graduationYear
     For Mentors: placedCompany, placedPosition, placementYear
                    location, contactVisibility (PUBLIC/ADMIN_ONLY)
     isApproved (mentor verification status)

2  mentors (Separate verified mentor profiles)
     id, fullName, email, phoneNumber
     linkedinProfile, placedCompany, placedPosition
     placementYear, departmentIds[]
     isActive

3  admins
     id, username (unique), password (hashed)
     fullName, email
     createdAt, lastLogin
     isActive

4  departments
     id, departmentName, departmentCode (CSE, IT, ECE, etc.)
     description, departmentGroup
     isActive

5  companies
     id, companyName (unique), description
     industry, website, logoUrl, headquarters
     interviewExperienceIds[]
     createdAt, createdById

6  interview_experiences
     id, studentName, rollNumber, department
     companyName, position, yearOfPlacement
     totalRounds, roundsJson (interview rounds data)
     result (PLACED/REJECTED), salary
     internOffered, hasBond, bondDetails
     tips, resources[]
     createdAt, updatedAt

7  placement_experiences (comprehensive format)
     Similar to interview_experiences
     More detailed round-by-round breakdown
     Enhanced metadata


 AUTHENTICATION & AUTHORIZATION



                        USER REGISTRATION FLOW                            


STUDENTS:
1. Select 'Student' role
2. Enter @gct.ac.in email (validated)
3. Receive OTP via email (6-digit code, 60s timeout)
4. Verify OTP
5. Complete registration form
6.  Auto-approved, can login immediately

MENTORS:
1. Select 'Mentor' role
2. Enter any valid email
3. Receive OTP via email
4. Verify OTP
5. Complete mentor details (company, position, LinkedIn, phone)
6. Choose contact visibility (PUBLIC or ADMIN_ONLY)
7.  Auto-approved (no admin verification needed)

ADMINS:
- Created by existing admins only
- No self-registration
- Username + password login


 API ENDPOINTS (75+ endpoints)


 Authentication (/api/auth)
   POST   /register              - User registration
   POST   /login                 - User login
   POST   /send-otp              - Send email OTP
   POST   /verify-otp            - Verify OTP
   GET    /user/{id}             - Get user by ID
   PUT    /user/{id}/profile     - Update profile
   GET    /mentors               - Get all mentors
   GET    /mentors/pending       - Get pending mentor approvals
   PUT    /mentors/{id}/approve  - Approve mentor
   DELETE /mentors/{id}/reject   - Reject mentor

 Companies (/api/companies)
   GET    /                      - Get all companies
   POST   /                      - Create company
   GET    /{id}                  - Get by ID
   GET    /name/{name}           - Get by name
   GET    /search?query=...      - Search companies
   GET    /exists?companyName=...- Check if exists

 Departments (/api/departments)
   GET    /                      - Get all departments
   GET    /{id}                  - Get by ID
   GET    /group/{group}         - Get by department group

 Experiences (/api/experiences)
   GET    /                      - Get all experiences
   POST   /                      - Create experience
   GET    /{id}                  - Get by ID
   PUT    /{id}                  - Update experience
   DELETE /{id}                  - Delete experience
   GET    /department/{id}       - Filter by department
   GET    /search/company?name=...- Search by company
   GET    /year/{year}           - Filter by year

 Admin (/api/admin)
   POST   /login                 - Admin login
   POST   /create                - Create new admin
   GET    /users                 - Get all users
   GET    /mentors               - Get all mentors
   PUT    /mentors/{id}          - Update mentor
   DELETE /users/{id}            - Delete user
